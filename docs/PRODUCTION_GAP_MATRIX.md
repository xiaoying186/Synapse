# Synapse Production Gap Matrix

This matrix tracks the gap between the current public baseline and the final
goal: all designed Synapse capabilities implemented and ready for production
use.

Status labels:

| Status | Meaning |
| --- | --- |
| production-ready | Implemented, tested, documented, and release-gated |
| alpha-usable | Works locally behind safety gates, but needs broader validation |
| guarded-preview | UI/API shape exists, real action remains gated or disabled |
| design-only | Design intent exists, implementation is not ready |

## Operating Rule

A capability can move toward production only when all of these are true:

1. It has a clear owner center: Taiheng, Zhishu, Xingtai, or Baigong.
2. It has an explicit safety boundary and denied-action list.
3. It has durable audit evidence for successful and denied actions.
4. It has rollback or recovery guidance for any high-risk mutation.
5. It is represented in `docs/CAPABILITY_MATRIX.md`.
6. It has at least one targeted automated verification command.
